# Data Hub

[К оглавлению](README.md)

Data Hub - подсистема данных ITS. Она отвечает за подключение источников, получение рыночной информации, нормализацию данных и предоставление единого API для остальных сервисов.

![Data Hub UI](img/data_ui.png)

## Назначение

Data Hub решает три группы задач:

- предоставляет модельеру визуальный интерфейс для просмотра данных;
- предоставляет backend API для Strategy Lab и GA Lab;
- изолирует код интеграций с внешними источниками от логики стратегий.

## Основные файлы

| Путь | Назначение |
| --- | --- |
| `services/data_backend/app/main.py` | FastAPI-приложение Data Backend |
| `ui/data-ui` | Vue-интерфейс Data Hub |
| `its/data_loader` | слой загрузчиков и адаптеров источников данных |
| `its/data_loader/t_invest_data_readers` | подключенный источник T-Invest |
| `its/data_loader/custom_bar/gold_bar` | построение custom gold bars |

## Доступные данные в базовой версии

Базовая версия подключает T-Invest как источник данных и предоставляет:

- российские акции;
- валютные инструменты;
- свечи OHLCV;
- информацию о тикерах;
- сектора, биржи, валюты, классы инструментов;
- дивиденды;
- custom gold bars, пересчитанные через золото.

## Разделы UI

### Котировки

Раздел предназначен для просмотра свечей выбранного инструмента.

Пользователь выбирает:

- класс инструмента, например `TQBR`;
- интервал свечей;
- дату начала;
- дату окончания;
- инструмент.

UI отображает:

- свечной график;
- последнюю цену;
- процентное изменение за период;
- число свечей;
- объем.

### Дивиденды

Раздел показывает дивидендную историю выбранной бумаги:

- дата объявления;
- последний день покупки;
- дата фиксации;
- дата выплаты;
- размер дивиденда;
- доходность;
- тип и регулярность выплаты.

Дивиденды используются не только для анализа, но и как входные данные для компонентов стратегии, например `DividendHistorySelector`.

### Инструменты

Раздел справочника акций. Пользователь может искать бумаги по названию или тикеру и видеть:

- FIGI;
- тикер;
- ISIN;
- название;
- сектор;
- биржу;
- валюту;
- страну риска;
- лот;
- торговый статус;
- доступность покупки и продажи.

### Валюты

Раздел валютных инструментов. Используется в том числе для `GLDRUB_TOM`, который нужен при построении custom gold bars.

### Custom gold-bar

Data Hub умеет строить альтернативные бары через стоимость золота. Пользователь задает:

- базовый инструмент;
- золотой инструмент, по умолчанию `GLDRUB_TOM`;
- количество золотых единиц;
- тип бара, например `GRAM`, `T_OUNCE`, `KG`, `T_OUNCE_400`;
- период и интервал.

Результат - OHLCV-бары, пересчитанные через стоимость выбранного количества золота.

## Data Backend API

Все запросы снаружи идут через gateway:

```text
/api/data/
```

Основные endpoints:

| Endpoint | Назначение |
| --- | --- |
| `GET /api/data/health` | проверка статуса |
| `GET /api/data/sources` | список источников |
| `GET /api/data/stocks` | список акций |
| `GET /api/data/currencies` | список валют |
| `GET /api/data/prices` | свечи OHLCV |
| `GET /api/data/custom-gold-bars` | custom gold bars |
| `GET /api/data/dividends` | дивиденды |
| `GET /api/data/docs` | Swagger |

## Поддерживаемые интервалы свечей

Data Backend принимает интервалы:

- `CANDLE_INTERVAL_1_MIN`;
- `CANDLE_INTERVAL_5_MIN`;
- `CANDLE_INTERVAL_15_MIN`;
- `CANDLE_INTERVAL_HOUR`;
- `CANDLE_INTERVAL_DAY`;
- `CANDLE_INTERVAL_WEEK`;
- `CANDLE_INTERVAL_MONTH`.

## Кэширование

Загрузчики T-Invest используют файловый кэш внутри `its/data` в контейнере. В Docker Compose эта директория вынесена в volume:

```text
t-invest-cache
```

Кэш снижает число обращений к внешнему API, ускоряет повторные тесты и позволяет переиспользовать уже загруженные периоды.

## Добавление нового источника данных

Новые интеграции добавляются в:

```text
its/data_loader
```

Рекомендуемый порядок:

1. Создать адаптер источника.
2. Нормализовать выходные поля к формату, который ожидает Data Backend.
3. Добавить use case, если требуется отдельная бизнес-логика.
4. Расширить Data Backend endpoint или добавить новый.
5. Обновить Data UI, если данные должны быть доступны пользователю визуально.

## Требования к данным для стратегий

Стратегии и тесты ожидают, что свечи содержат как минимум:

| Поле | Назначение |
| --- | --- |
| `time` | дата и время свечи |
| `ticker` | тикер инструмента |
| `figi` | FIGI |
| `open` | цена открытия |
| `high` | максимум |
| `low` | минимум |
| `close` | цена закрытия |
| `volume` | объем |
| `is_complete` | признак завершенной свечи |

Для дивидендных селекторов нужны:

| Поле | Назначение |
| --- | --- |
| `ticker` | тикер |
| `last_buy_date` | последний день покупки для получения дивиденда |
